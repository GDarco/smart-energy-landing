# Документація лінтингу проєкту Smart-Energy

## Обраний інструмент
Для проєкту обрано **Checkstyle** — інструмент статичного аналізу коду для Java.
**Причина вибору:** Відповідність галузевим стандартам (Google/Sun Java Style) та висока гнучкість налаштувань.

## Базові правила
1. `MethodName`: Методи повинні починатися з маленької літери (camelCase).
2. `NeedBraces`: Обов'язкове використання фігурних дужок навіть для однорядкових `if/else`.
3. `WhitespaceAround`: Перевірка наявності пробілів навколо операторів для кращої читабельності.

## Інструкція з запуску
У IntelliJ IDEA: `Analyze` -> `Checkstyle` -> `Run`.
